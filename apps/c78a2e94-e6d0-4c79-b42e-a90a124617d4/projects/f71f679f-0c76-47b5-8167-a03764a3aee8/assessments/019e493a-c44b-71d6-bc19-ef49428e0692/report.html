Core Business Workflows

This application serves banking clients that need exchange-rate retrieval and amount conversion for supported currency pairs. The core workflow is rate lookup and conversion backed by current market rates stored in SQL Server.

Domain Entities

EntityService / Bounded ContextDescriptionKey Relationships
CurrencyPairCurrency PricingRepresents an active base/quote pair offered by the serviceOne pair has many historical exchange rates
ExchangeRateCurrency PricingRepresents a sourced market rate effective over a date intervalBelongs to one currency pair
CurrencyRateDataCurrency API ContractResponse contract returned to clients for rate queriesDerived from CurrencyPair + latest ExchangeRate

Service-to-Domain Mapping

ServiceDomain ContextOwned EntitiesExternal Dependencies
ZavaCurrencyServiceCurrency PricingCurrencyPair, ExchangeRate, CurrencyRateData contractSQL Server

Primary Workflows

Workflow 1: Retrieve latest exchange rate

A client requests GetRate with source and target currencies. The service normalizes/validates currency codes, loads the latest non-expired active rate for the pair, and returns CurrencyRateData. If no rate is found, it returns a fault response.

Workflow 2: Convert amount between currencies

A client requests ConvertAmount with amount and pair. The service validates non-negative amount, reuses the rate lookup flow, computes rounded converted value, and returns a decimal result.

Workflow 3: List supported currencies

A client requests GetSupportedCurrencies. The service queries active pairs, unions base and quote codes, sorts them, and returns the list.

Cross-Service Data Flows

No cross-service aggregation is present. Data is read from a single SQL Server source and transformed into service contract responses.

Business Workflow Sequence

sequenceDiagram participant Client participant Service as "CurrencyService" participant DB as "SQL Server" Client->>Service: Request conversion (amount, from, to) Service->>Service: Validate amount and currency codes Service->>DB: Find latest active non-expired rate alt Rate exists DB-->>Service: Current exchange rate Service->>Service: Calculate rounded converted amount Service-->>Client: Converted amount else Rate missing DB-->>Service: No result Service-->>Client: Fault - no exchange rate found end

Business Rules & Decision Logic

  • Currency code rule: input currency must be exactly 3 characters after trimming.
  • Amount rule: conversion amount must be non-negative.
  • Rate eligibility rule: only active pairs and non-expired rates are valid.
  • Conversion rule: converted value is rounded to 4 decimals using away-from-zero midpoint rounding.
